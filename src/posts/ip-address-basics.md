# ::1은 무엇일까? 숫자로 이루어진 주소, IP의 체계 이해하기

최근 댓글 기능을 개발하면서 로컬 환경에서 테스트를 진행하던 중, 데이터베이스에 저장된 작성자의 IP 주소가 `127.0.0.1`이 아닌 `::1`로 찍히는 것을 보고 잠시 당황했던 기억이 있습니다. 분명 익숙한 숫자의 나열이어야 할 IP 주소가 왜 이렇게 낯선 모습으로 나타난 것일까요?

단순히 "로컬호스트를 뜻한다"고 알고 넘어갈 수도 있었지만, 이번 기회에 IP 주소가 어떤 체계로 이루어져 있고 왜 다양한 형태를 가지는지 깊이 있게 들여다보기로 했습니다.

---

## 1. IP란 무엇인가: 인터넷 공간의 우편번호

IP(Internet Protocol) 주소는 말 그대로 인터넷 프로토콜을 사용하는 네트워크 상의 기기들이 서로를 식별하고 통신하기 위해 사용하는 '고유 주소'입니다. 현실 세계의 집 주소나 우편번호와 같은 역할을 수행하죠.

IP 주소는 크게 두 가지 핵심 역할을 합니다.
- **식별(Identification)**: 어떤 기기인지 구별합니다.
- **위치(Location)**: 네트워크 상에서 어디에 있는지 경로를 찾을 수 있게 합니다.

![IP 주소의 개념 체계](/public/images/ip_address_concept.png)

위 그림처럼 전 세계의 수많은 서버와 개인 기기들은 저마다의 IP 주소를 가지고 거대한 그물망처럼 연결되어 정보를 주고받습니다.

---

## 2. IPv4와 IPv6: 왜 ::1이 나타났을까?

우리가 흔히 보는 `192.168.0.1` 같은 형태는 **IPv4**입니다. 32비트 체계로 약 43억 개의 주소를 만들 수 있지만, 인터넷 기기가 폭발적으로 늘어나면서 주소가 고갈되는 문제에 직면했습니다.

이를 해결하기 위해 등장한 것이 **IPv6**입니다. 128비트 체계로 사실상 무한대에 가까운 주소를 제공하죠. 내 PC에서 보았던 `::1`은 바로 IPv6 체계에서의 로컬호스트(Loopback 주소)였습니다. 

- **IPv4 로컬호스트**: `127.0.0.1`
- **IPv6 로컬호스트**: `::1` (0:0:0:0:0:0:0:1의 축약형)

최근 로컬 개발 환경(Next.js, Node.js 등)이나 네트워크 설정이 IPv6를 우선순위로 두는 경우가 많아지면서, 자연스럽게 `::1`을 더 자주 마주하게 된 것이었습니다.

---

## 3. 공공 IP와 사설 IP: 우리만의 비밀 주소

모든 기기가 전 세계에 공개된 주소를 가질 필요는 없습니다. 여기서 **공공(Public) IP**와 **사설(Private) IP**의 개념이 나옵니다.

- **공공 IP**: 인터넷 서비스 제공업체(ISP)로부터 할당받은, 전 세계에서 유일한 주소입니다.
- **사설 IP**: 공유기 내부 망에서 우리 기기들끼리만 사용하는 주소입니다. (`192.168.x.x` 등)

공유기(NAT)가 이 사이에서 다리 역할을 해주어, 여러 기기가 하나의 공공 IP를 공유하면서도 각자의 사설 IP로 안전하게 통신할 수 있게 돕습니다.

---

## 실전 적용: IP 유효성 검사해보기

학습한 내용을 바탕으로, 서비스에서 입력받은 IP 주소가 유효한지, 그리고 로컬 주소인지를 판별하는 간단한 유틸리티를 작성해 보았습니다.

```typescript
/**
 * IP 주소 관련 유틸리티 (TypeScript)
 */

export function isValidIP(ip: string): boolean {
  // IPv4 및 IPv6 정규식을 활용한 검사
  const ipv4Regex = /^(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\.(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\.(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\.(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)$/;
  const ipv6Regex = /^(([0-9a-fA-F]{1,4}:){7,7}[0-9a-fA-F]{1,4}|...)$/; // (생략된 IPv6 패턴)
  
  return ipv4Regex.test(ip) || ipv6Regex.test(ip);
}

export function isLocalhost(ip: string): boolean {
  return ip === '127.0.0.1' || ip === '::1' || ip === 'localhost';
}
```

이 로직을 통해 내가 마주했던 `::1`이 단순한 에러가 아닌, 정교하게 설계된 현대 네트워크 체계의 일부임을 코드로도 확인할 수 있었습니다.

## 깨달음과 성장

단순히 숫자의 나열이라고만 생각했던 IP 주소 안에는 주소 고갈 문제를 해결하기 위한 인류의 고민(IPv6)과 효율적인 자원 활용을 위한 구조(NAT/사설 IP)가 담겨 있었습니다. 

`::1`을 처음 보았을 때의 당혹감은 이제 네트워크 기본기에 대한 확신으로 바뀌었습니다. 앞으로는 사용자 식별이나 보안 로직을 짤 때, IPv4뿐만 아니라 IPv6 환경까지 폭넓게 고려해야 한다는 점을 잊지 말아야겠습니다. 기초가 튼튼해야 더 높은 곳을 바라볼 수 있다는 평범한 진리를 다시 한번 되새기게 된 소중한 경험이었습니다.
