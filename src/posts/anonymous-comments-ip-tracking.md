# 비로그인 사용자를 위한 익명 댓글 시스템: IP 식별과 보안

로그인 없이 누구나 댓글을 달 수 있게 하는 것은 사용자 참여를 이끄는 좋은 방법입니다. 하지만 무분별한 도배나 악성 댓글을 방지하기 위해서는, 최소한 작성자를 식별할 수 있는 수단이 필요합니다. 이번 포스팅에서는 **Next.js의 `headers()`를 이용해 클라이언트 IP를 추출하고 활용하는 방법**을 다룹니다.

---

## 1. 서버 측에서 클라이언트 IP 추출하기

Next.js의 Server Actions나 Server Components 환경에서는 `next/headers` 모듈을 통해 클라이언트의 헤더 정보에 직접 접근할 수 있습니다. 특히 프록시나 로드 밸런서를 거치는 실제 배포 환경에서는 `x-forwarded-for` 헤더를 확인하는 것이 중요합니다.

```typescript
// src/app/actions/comments.ts
import { headers } from 'next/headers';

export async function submitComment(formData: FormData) {
  // 헤더 리스트 가져오기
  const headerList = await headers();
  
  // 프록시 환경 고려: x-forwarded-for, x-real-ip 순으로 확인
  const ip = headerList.get('x-forwarded-for')?.split(',')[0] 
          || headerList.get('x-real-ip') 
          || '127.0.0.1';

  console.log(`클라이언트 IP: ${ip}`);

  // 데이터베이스에 IP와 함께 댓글 저장
  // await addComment({ ..., ip });
}
```

## 2. 왜 IP를 수집해야 할까요?

익명 댓글 시스템에서 IP 수집은 다음과 같은 역할을 합니다:

- **도배 방지 (Rate Limiting):** 특정 IP에서 짧은 시간 동안 너무 많은 댓글이 올라오지 않도록 차단하거나 제한할 수 있습니다.
- **최소한의 식별:** 문제 발생 시 차단 등의 조치를 취할 수 있는 유일한 식별자가 됩니다.
- **신뢰도 향상:** 작성자 본인 임을 간접적으로 증명하거나, 중복 투표 방지 등에 활용할 수 있습니다.

## 3. 개인정보 보호와 주의사항

IP 주소는 개인정보로 분류될 수 있으므로 처리에 주의가 필요합니다.

1. **최소 수집 원칙:** 도배 방지라는 명확한 목적을 가지고 수집하며, 필요 이상의 정보를 요구하지 않습니다.
2. **저장 보안:** 데이터베이스 접근 권한을 엄격히 관리하고, 외부로 유출되지 않도록 암호화하거나 마스킹 처리를 고려할 수 있습니다. (예: `123.***.***.456`)
3. **개인정보 처리방침:** 블로그에 댓글 작성 시 IP가 수집된다는 사실을 명시하는 것이 투명성 차원에서 좋습니다.
