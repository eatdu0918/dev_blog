# 엔디안(Endianness): 바이트 오더에 대한 이해와 네트워크 디버깅 경험

컴퓨터가 메모리에 멀티 바이트 데이터(예: 4바이트 정수형)를 저장할 때 바이트를 배열하는 순서를 **엔디안(Endianness)**이라고 합니다. 

학부 전공 수업 때 이런 개념이 있다는 정도만 듣고 넘어갔다가, 실무에서 브라우저 기반 바이너리 데이터(`ArrayBuffer`, `DataView`)를 다루며 제대로 이해하지 못해 뼈아픈 디버깅 시간을 보냈던 적이 있습니다. 그때의 경험을 되짚어보며 두 가지 엔디안 방식의 차이를 정리해 보고자 합니다.

---

## 🔍 빅 엔디안 (Big-Endian) vs 리틀 엔디안 (Little-Endian)

16진수로 표현된 4바이트 정수 `0x12345678`가 메모리에 저장되는 방식을 기준으로 살펴보겠습니다.

*(참고: `0x12`가 최상위 바이트(MSB)이고, `0x78`이 최하위 바이트(LSB)입니다.)*

### 1. 빅 엔디안 (Big-Endian)
- **방식**: 큰 단위(MSB)부터 낮은 메모리 주소에 차례대로 저장합니다. 사람이 숫자를 읽는 자연스러운 방향과 일치합니다.
- **저장 모습**: 
  | 주소 | 0x00 | 0x01 | 0x02 | 0x03 |
  | :--- | :--- | :--- | :--- | :--- |
  | 값   | `12` | `34` | `56` | `78` |
- **특징**: 디버깅 시 사람이 메모리를 순서대로 읽기 매우 직관적입니다. 네트워크 프로토콜에서 주로 사용됩니다.

### 2. 리틀 엔디안 (Little-Endian)
- **방식**: 작은 단위(LSB)부터 낮은 메모리 주소에 거꾸로 저장합니다. 
- **저장 모습**: 
  | 주소 | 0x00 | 0x01 | 0x02 | 0x03 |
  | :--- | :--- | :--- | :--- | :--- |
  | 값   | `78` | `56` | `34` | `12` |
- **특징**: 사람이 읽기엔 직관적이지 않지만, 컴퓨터의 산술 논리 장치(ALU)가 계산을 수행하기에는 설계가 더 최적화되어 있습니다. 일반적인 개인용 컴퓨터(Intel x86 등)에서 널리 사용됩니다.

---

## 🌐 네트워크 통신에서의 문제 발생

과거 사이드 프로젝트 환경에서 웹 브라우저 클라이언트와 Node.js 서버 간에 바이너리 덩어리(`ArrayBuffer`)를 직접 주고받는 기능을 개발한 적이 있었습니다. 로컬 환경의 제 PC 안에서만 테스트를 진행했을 때는 데이터가 100% 정상적으로 주고받아져서 완벽하게 구현했다고 생각했습니다.

문제가 발생한 것은 클라이언트는 제 윈도우 PC(리틀 엔디안), 서버는 클라우드의 특정 UNIX 기반 인스턴스(빅 엔디안)로 분리 배포된 후였습니다. 클라이언트에서 `1` (`0x00000001`) 이라는 4바이트 정수 코드를 보냈는데, 서버 로그에는 자꾸만 `16777216` (`0x01000000`) 이라는 알 수 없는 값이 수신된 것으로 찍혔습니다.

원인을 파악하기 위해 몇 시간 동안 코드를 뒤집어엎고 나서야, 운영체제나 CPU 아키텍처에 따라 엔디안 방식이 다를 수 있다는 사실을 떠올리게 되었습니다. 클라이언트(리틀 엔디안)가 던진 4바이트 덩어리를 서버(빅 엔디안)가 역순으로 읽어 들이며 데이터가 완전히 변질된 것이었습니다.

## 🛠️ 해결과 회고: 네트워크 바이트 오더

이러한 시스템 간의 불일치를 막기 위해, 인터넷 프로토콜 표준은 네트워크로 데이터를 송출할 때 **'네트워크 바이트 오더(Network Byte Order)'**를 따라 무조건 **빅 엔디안(Big-Endian)**으로 통일하여 변환하도록 규정하고 있다는 사실을 뒤늦게 알게 되었습니다.

JavaScript의 `DataView` API를 찾아보니, 값을 읽거나 쓸 때 세 번째 인자로 엔디안을 지정할 수 있었습니다 (`true`면 리틀 엔디안, `false`나 생략하면 빅 엔디안). 이를 활용하여 통신 전 양측의 바이트 오더를 명시적으로 맞추는 작업을 추가하고 나서야 버그가 깔끔히 해결되었습니다.

```javascript
// 클라이언트 측 (전송 시 빅 엔디안으로 강제 지정)
const buffer = new ArrayBuffer(4);
const view = new DataView(buffer);
// 1이라는 값을 0번 오프셋에 빅 엔디안으로 쓴다.
view.setInt32(0, 1, false); 
```

평소 고수준(High-level) JSON API 통신 등에 의존해 네트워크 통신을 쉽게 처리해오다가, 로우 레벨에 가까운 바이너리 패킷을 브라우저와 노드 환경에서 직접 구성해 보면서 CS 기초 지식이 왜 중요한지 실감하는 계기가 되었습니다.
