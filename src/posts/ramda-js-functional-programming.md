# Ramda.js: 명령형의 늪에서 탈출하여 함수형으로

![Ramda.js 컨셉 다이어그램](/public/images/ramda-js-functional.png)

프론트엔드 개발을 하다 보면 API에서 내려온 복잡한 객체 배열을 화면에 맞게 가공해야 하는 일이 정말 많다. 처음에는 별생각 없이 `for` 문을 돌리고 외부 변수를 수정하는 방식으로 코드를 짰다. 하지만 요구사항이 늘어날수록 코드는 점점 `if`, `else`의 미로가 되었고, "도대체 이 변수는 어디서 누가 바꾼 거지?"라는 의문과 함께 디버깅에 시간을 쏟는 나를 발견했다.

그러던 중 **Ramda.js**라는 라이브러리를 만났다. 단순히 기능을 제공하는 도구를 넘어, 데이터를 바라보는 관점 자체를 바꿔준 이 특별한 라이브러리를 통해 내가 느꼈던 '함수형 프로그래밍'의 매력을 정리해 보려고 한다.

---

## 1. 커링(Currying): "함수를 조각내는 즐거움"

Ramda의 모든 함수는 기본적으로 **커링(Currying)**되어 있다. 즉, 인자를 한 번에 다 주지 않아도 된다. 인자가 부족하면 '나머지를 받아줄 준비가 된 새로운 함수'를 반환한다.

### 내가 시도해 본 변화
기존에는 특정 속성값을 추출하기 위해 매번 인자를 받는 함수를 새로 만들었다. 하지만 Ramda를 사용하니 기존 함수를 재조합하여 아주 손쉽게 특화된 함수를 만들어낼 수 있었다.

```javascript
import R from 'ramda';

// 'name' 속성을 가져오는 범용 함수
const getName = R.prop('name'); 

// 나중에 객체만 주면 바로 이름이 튀어나온다.
const user = { name: 'Alice', age: 25 };
console.log(getName(user)); // 'Alice'
```

---

## 2. 파이프(Pipe): "데이터의 흐름을 설계하다"

Ramda의 진가는 여러 연산을 연결할 때 드러난다. `R.pipe`는 여러 함수를 순서대로 연결하여 하나의 거대한 '파이프라인'을 만든다.

데이터가 왼쪽에서 오른쪽으로 흐르며 각 함수를 통과하는 모습은 마치 조립 라인과 같았다. 무엇보다 좋은 점은 데이터(변수명)를 명시하지 않고도 로직을 설계할 수 있는 **포인트 프리(Point-free)** 스타일이 가능하다는 것이다.

### 실전 예제
사용자 목록에서 성인만 골라 이름을 추출하고 알파벳 순으로 정렬하는 로직을 구현해 보았다.

```javascript
const processUsers = R.pipe(
  R.filter(R.propSatisfies(age => age >= 20, 'age')),
  R.map(R.prop('name')),
  R.sortBy(R.identity)
);
```

이 코드를 처음 마주했을 때의 충격을 잊을 수 없다. "어떻게 할 것인가(How)"에 대한 명령(for, if)은 사라지고, "무엇을 할 것인가(What)"에 대한 흐름만 남았다.

---

## 3. 불변성(Immutability): "안전한 코드의 기초"

Ramda는 원본 데이터를 절대 건드리지 않는다. 모든 연산은 복사본을 만들어 새로운 데이터를 반환한다. 처음에는 "성능에 문제가 있지 않을까?" 걱정도 했지만, 상태 변화를 추적하기 쉬워지고 예기치 못한 사이드 이펙트가 사라지면서 얻는 안정감이 훨씬 컸다.

---

## 💡 마치며: 도구를 넘어 철학을 배우다

Ramda.js를 학습하며 가장 크게 깨달은 것은, 함수형 프로그래밍이 단순히 '깔끔한 코드'를 만드는 기술이 아니라는 점이다. 그것은 **상태를 직접 관리하려는 욕심을 내려놓고, 데이터를 흐름(Flow)으로 인식하는 철학**에 가깝다.

물론 모든 곳에 Ramda를 써야 하는 것은 아니다. 때로는 단순한 `for` 문이 가독성이 좋을 때도 있다. 하지만 복잡한 데이터 변환이 얽힌 상황에서 Ramda가 보여준 '조합의 마법'은 나에게 프로그래밍의 새로운 지평을 열어주었다. 이제는 코드를 짤 때 "어떤 데이터를 어떻게 바꿀까"가 아니라 "어떤 함수들을 이어서 이 파이프를 완성할까"를 먼저 고민하게 된다.

함수형 세계로의 첫 발걸음은 낯설었지만, 그 끝에서 만난 선언적인 코드의 아름다움은 내가 더 좋은 개발자로 성장하고 있다는 확신을 주기에 충분했다.
