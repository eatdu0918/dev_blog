# 개발을 모르는 건 아니지만 자동화는 막막해서, n8n을 파헤쳐보기로 했다

## "뭔가 자동화는 하고 싶은데, 코드 짤 엄두가 안 났다"

매일 아침 루틴처럼 하는 일들이 있다. 특정 웹사이트를 확인하고, 변경 사항이 있으면 정리해서 메신저로 보내는 일 같은 것들이다. 개발자라면 당연히 '스크립트 하나 짜서 돌리면 끝 아니야?'라고 생각할 수 있겠지만, 막상 시작하려니 생각할 게 너무 많았다. 어디서 실행해야 하지? API 연동은 또 어떻게 하지?

그러던 중 우연히 익숙한 로고와 함께 'n8n'이라는 이름을 접했다. 처음에는 또 다른 복잡한 프레임워크인 줄 알고 겁이 났지만, 노코드(No-code) 혹은 로우코드(Low-code) 기반의 자동화 도구라는 설명을 보고 호기심이 생겼다. "진짜 코드 한 줄 없이도 내가 원하는 일을 시킬 수 있을까?"라는 의문과 함께 검색창을 열었다.

## 도대체 n8n이 뭐야? (학습자의 조사 일지)

가장 먼저 찾아본 것은 n8n의 정체였다. 자료를 조사하며 내가 이해한 n8n은 '서비스와 서비스를 연결해주는 지능형 다리'였다. 

![n8n 학습 개념도](/public/images/n8n-learning.png)

인터넷에 나와 있는 방대한 정보들을 나름의 방식으로 요약해 보니 몇 가지 생소하지만 중요한 키워드를 발견할 수 있었다.

1.  **노드(Node) 위주의 세상**: n8n에서는 모든 동작이 '노드'라는 블록에서 시작된다는 걸 알게 되었다. Slack 노드, Discord 노드, 심지어는 JSON 데이터를 만지는 노드까지. 이 노드들을 화면 위에서 선으로 연결하기만 하면 그게 곧 자동화 프로그램이 된다고 한다.
2.  **데이터의 통로**: 노드와 노드를 연결하는 선은 단순히 순서를 정하는 게 아니었다. 앞선 노드가 가져온 데이터를 다음 노드로 넘겨주는 '혈관' 같은 역할이다. 모든 데이터는 JSON이라는 형식으로 흐른다는 점이 개발자인 나에게는 그나마 친숙하게 다가왔다.
3.  **내 서버에 직접? (Self-hosting)**: 많은 자동화 서비스들이 월 정액을 요구하거나 보안 문제를 안고 있지만, n8n은 직접 내 로컬 환경이나 서버에 설치해서 쓸 수 있다는 게 큰 특징이었다. (물론 아직 설치는 안 해봤지만, 가능하다는 사실만으로도 든든했다.)

## 아직은 상상만 해보는 워크플로우

자료를 읽다 보니 머릿속으로 '내가 이걸로 뭘 할 수 있을까?'를 그려보게 되었다. 예를 들어, 내가 쓴 블로그 글을 자동으로 요약해서 트위터에 올려주는 흐름 같은 것들이다.

이해를 돕기 위해, n8n이 내부적으로 데이터를 어떻게 넘겨줄지를 아주 간단한 코드로 흉내 내보았다. 실제 n8n을 써본 건 아니지만, "이런 식으로 데이터가 바통 터치를 하겠구나"라고 추측하며 짜 본 코드다.

```typescript
// n8n이 이런 식으로 동작하지 않을까? (상상하며 짠 추측 코드)

async function imaginaryN8nWorkflow(initialData: any) {
  // 1. 데이터를 가져오는 노드 (추측)
  const inputData = await fetchSourceNode(initialData); 
  
  // 2. 데이터를 가공하는 노드 (추측)
  const processedData = await transformNode(inputData);
  
  // 3. 최종적으로 어딘가에 보내는 노드 (추측)
  await sendDestinationNode(processedData);
}
```

조사를 해보니 n8n의 실제 인터페이스는 훨씬 정교하겠지만, 근본적으로는 이런 '순차적인 데이터 흐름'을 만드는 게 핵심이라는 점을 어렴풋이 깨달았다.

## 마무리하며: 다음 단계는 '실전'이다

오늘 하루 동안 n8n에 대해 조사하며 느낀 점은, 내가 그동안 '자동화'라는 벽을 너무 높게만 생각했다는 것이다. 모든 것을 처음부터 끝까지 코드로 구현해야 한다는 강박에서 벗어나, 이미 잘 만들어진 도구를 활용하는 것도 훌륭한 전략이 될 수 있겠다는 생각이 들었다.

물론 아직 직접 대시보드를 열어본 적도 없고, 노드 하나 연결해 본 적 없는 초보 중의 초보 상태다. 하지만 이제 무엇이 중요한 개념인지는 알 것 같다. 다음에는 자료 조사를 넘어, 실제로 내 컴퓨터에 n8n을 설치하고 간단한 워크플로우 하나를 직접 돌려보는 성공 경험을 쌓아보고 싶다.
