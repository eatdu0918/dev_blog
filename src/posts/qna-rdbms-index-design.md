## 핵심 요약

복합 인덱스 컬럼 순서는 **(1) 등호 조건 → (2) 정렬/범위 조건 → (3) 선택도 높은 순**이라는 우선순위로 정합니다. 그리고 인덱스는 공짜가 아니므로 **쓰기 비용**까지 고려해 추가/제거 의사결정을 합니다.

## 컬럼 순서를 정하는 원칙

### 1. WHERE 절의 등호(=) 조건이 가장 앞

`(a, b, c)` 인덱스는 `a = ?`만 있어도 사용 가능, `a = ? AND b = ?`도 사용 가능, 그러나 `b = ?`만 있으면 사용 불가입니다. 즉 **앞 컬럼부터 매칭**되어야 인덱스를 탑니다.

### 2. 범위(BETWEEN, >, <)와 정렬은 등호 뒤에

범위 조건이 들어가면 그 뒤 컬럼은 인덱스 정렬 효과를 못 봅니다. 그래서 등호 조건이 앞, 범위/정렬이 뒤에 오는 게 원칙입니다.

```sql
-- (status, created_at) 인덱스가 가장 효율적
SELECT * FROM orders
WHERE status = 'PAID'
ORDER BY created_at DESC
LIMIT 20;
```

### 3. 선택도(distinct ratio)가 높은 컬럼을 앞에

선택도란 "전체 row 중 그 값으로 얼마나 좁혀지는가"입니다. `gender`처럼 2가지 값밖에 없는 컬럼을 앞에 두면 인덱스 효과가 약합니다. 단, **앞 두 원칙이 우선**입니다. 선택도만 보고 등호 조건을 뒤로 보내는 건 잘못된 선택입니다.

## EXPLAIN으로 검증

설계는 가설일 뿐이고, 검증은 EXPLAIN으로 합니다.

- `type`: `const`/`ref`/`range`가 좋고, `ALL`(full scan)이면 인덱스를 못 탔다는 뜻.
- `key`: 실제로 사용된 인덱스. NULL이면 안 탄 것.
- `rows`: 옵티마이저가 추정한 스캔 row 수. 실제와 큰 차이가 나면 통계 갱신 필요.
- `Extra`의 `Using index`(커버링 인덱스), `Using filesort`(추가 정렬 발생), `Using temporary`(임시 테이블 발생)는 모두 신호입니다.

## 인덱스를 함부로 추가하면 안 되는 이유

인덱스는 **읽기를 빠르게 하지만 쓰기를 느리게 만듭니다.**

- 모든 INSERT/UPDATE/DELETE가 인덱스 트리를 갱신해야 합니다.
- 인덱스 N개면 쓰기 비용도 대략 N배 가까이 증가합니다(컬럼이 인덱스에 포함된 경우).
- 인덱스 자체가 디스크 공간을 차지합니다. 큰 테이블의 인덱스는 base 테이블만큼 큰 경우도 흔합니다.
- 옵티마이저의 선택지가 많아지면 잘못된 인덱스를 고를 위험도 커집니다.

그래서 저는 인덱스를 추가할 때 **이 인덱스로 사라지는 비싼 쿼리가 무엇인지** 명확히 정합니다. "혹시 모르니"는 인덱스를 만들 사유가 안 됩니다.

## 자주 묻는 디테일

- **커버링 인덱스**: SELECT 절 컬럼까지 인덱스에 포함되면 테이블을 안 읽어도 되어 매우 빠름. 좁은 범위의 핵심 쿼리에 쓰면 효과가 큽니다.
- **함수 기반 인덱스**: `WHERE LOWER(email) = ?`처럼 함수를 씌우면 일반 인덱스를 못 탑니다. 함수 기반 인덱스를 만들거나 데이터 자체를 정규화해 저장.
- **NULL 처리**: 어떤 DB는 NULL이 인덱스에 포함되고, 어떤 DB는 빠집니다. 옵티마이저 동작과 함께 확인.
- **카디널리티 폭주**: `WHERE status IN (...)`에서 IN 리스트가 매우 길면 인덱스 효율이 떨어질 수 있어, 임시 테이블 join이 더 빠른 경우도 있습니다.

## 면접 follow-up

- "Slow query를 만났을 때 첫 단계는?" → EXPLAIN ANALYZE로 실제 실행 계획 확인 → 통계 갱신 여부 → 인덱스 검토 → 쿼리 재작성 순.
- "쓰기가 매우 많은 테이블에서 인덱스가 너무 많다면?" → 사용 빈도(`pg_stat_user_indexes`, `sys.schema_unused_indexes`)를 보고 사용 안 되는 인덱스 정리.
- "B-Tree 외 다른 인덱스 타입은?" → Hash, GIN, GiST, BRIN. 각각 적합한 워크로드(전문 검색, 시계열 등)를 답할 수 있으면 좋습니다.
