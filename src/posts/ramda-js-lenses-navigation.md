# 중첩된 불변 객체의 선언적 관리: Ramda.js Lenses의 활용

함수형 프로그래밍(FP)을 공부하다 보면 가장 먼저 마주하는 거대한 벽이 바로 '불변성(Immutability)'이다. 데이터를 직접 수정하지 않고 매번 복사해서 새로운 객체를 만드는 과정은 안전하지만, 객체의 깊이가 깊어질수록 코드가 급격히 지저분해지는 부작용을 낳기도 한다.

오늘은 이러한 불편함을 해결하기 위해 학습한 **Ramda.js의 Lenses(렌즈)** 개념을 정리해본다. 특정 프로젝트의 경험이라기보다, 복잡한 데이터를 다루는 더 나은 방식을 찾아 떠난 순수한 기술 탐구의 기록이다.

---

## 😫 왜 '렌즈'가 필요했을까?

자바스크립트에서 중첩된 객체를 불변하게 수정하려면 보통 스프레드 연산자(`...`)를 사용한다. 하지만 아래와 같이 3단계만 깊어져도 코드는 읽기 힘들어진다.

```javascript
/* 일반적인 불변 객체 수정 방식 */
const updatedData = {
  ...originalData,
  nested: {
    ...originalData.nested,
    target: {
      ...originalData.nested.target,
      value: 'new value'
    }
  }
};
```

이 코드는 실수의 여지가 많고, 무엇보다 "어떤 값을 바꾸려 하는가"라는 핵심 의도가 코드 뭉치 속에 파묻혀버린다. 렌즈는 바로 이 지점에서 **"어디를 바라볼 것인가"**를 데이터 구조와 분리하여 선언적으로 정의하게 해준다.

---

## 🔍 Lens: Getter와 Setter의 우아한 결합

함수형 프로그래밍에서 렌즈는 단순한 유틸리티를 넘어, 특정 데이터 구조를 읽고(View) 쓰는(Set) 규칙을 담은 하나의 추상화된 객체다.

![Ramda Lenses 개념 시각화](/public/images/ramda_js_lenses_concept.png)

Ramda에서는 `R.lensPath`, `R.lensProp` 등을 통해 렌즈를 생성할 수 있다. 학습하며 깨달은 가장 큰 장점은 **컴포지션(Composition)**이 가능하다는 점이다.

### 기초 사용법

```javascript
import * as R from 'ramda';

const data = { a: { b: { c: 1 } } };

// 경로 정의 (추상화)
const cLens = R.lensPath(['a', 'b', 'c']);

// 1. 읽기 (view)
R.view(cLens, data); // 1

// 2. 쓰기 (set) - 원본은 불변, 새 객체 반환
const newData = R.set(cLens, 2, data); // { a: { b: { c: 2 } } }

// 3. 변환 (over) - 함수를 적용하여 수정
const incrementedData = R.over(cLens, R.inc, data); // { a: { b: { c: 2 } } }
```

## 💡 학습하며 얻은 인사이트

### 1. 데이터 구조와 로직의 분리
렌즈를 한 번 정의해두면, 나중에 데이터 구조가 `['a', 'b', 'c']`에서 `['root', 'value']`로 바뀌더라도 렌즈 정의부만 수정하면 된다. 이를 사용하는 수많은 비즈니스 로직(view, set, over)은 수정할 필요가 없다.

### 2. 안전한 접근 (Safe Navigation)
직접 경로를 찾아 들어갈 때는 중간 단계가 `null`인 경우 `Uncaught TypeError`를 만날까 두렵지만, Ramda 렌즈는 내부적으로 안전하게 경로를 탐색한다.

### 3. 선언적인 코드 디자인
"첫 번째 필드를 복사하고..."라는 명령 위주의 사고방식에서 "이 경로의 값을 증가시킨다"는 결과 위주의 사고방식으로 코드를 짤 수 있게 되었다.

## 맺으며

Ramda Lenses를 공부하며 느낀 점은, 함수형 프로그래밍의 도구들이 단순히 코드를 짧게 만드는 기술이 아니라 **"적절한 제약 안에서 더 큰 자유를 주는 방식"**이라는 것이다. 

불변성을 지키는 것이 더 이상 고통스러운 반복 작업이 아니라는 것을 깨달은 것만으로도 큰 수확이었다. 앞으로 마주할 수많은 복잡한 상태 관리 로직에서도 이 '렌즈'라는 조준경을 통해 더 명확하고 견고한 코드를 짜보려 한다.
