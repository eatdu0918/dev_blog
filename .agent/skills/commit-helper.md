# 커밋 도우미 (Commit Helper Skill)

이 스킬은 사용자의 명령에 따라 현재의 프로젝트 변경 사항을 작업 단위로 분석하고, 명확한 커밋 목록과 제목을 제안하는 능력을 제공합니다.

## 1. 변경 사항 분석 (Analyze Changes)
- `git diff` 또는 파일 변경 이력을 확인하여 수정된 파일 목록을 파악합니다.
- 각 파일의 변경 내용이 프로젝트의 어떤 영역(예: `src/posts`, `src/components`, `package.json` 등)에 속하는지 분류합니다.

## 2. 작업 단위 그룹화 (Group Changes)
- 변경 사항들을 논리적인 기능 단위로 묶습니다.
- **예시 그룹화**:
    - 블로그 포스팅 마크다운 추가 + 해당 포스팅에 필요한 이미지 자산 (`src/posts/*.md` + `public/assets/*`)
    - 컴포넌트 리팩토링 + 관련 테스트 코드 수정 (`src/components/*` + `test/components/*`)
    - 의존성 업그레이드 및 설정 변경 (`package.json` + `.env.example`)

## 3. 커밋 목록 제안 (Suggest Commit List)
- 그룹화된 각 작업 단위에 대해 `commit-guidelines.md`에 정의된 형식에 맞춰 커밋 제목을 생성합니다.
- 사용자에게 다음과 같은 형식으로 제시합니다:
    ```markdown
    ### 📦 커밋 제안 목록
    | 순번 | 커밋 제목   | 포함된 파일                           |
    | :--- | :---------- | :------------------------------------ |
    | 1    | `docs: ...` | `src/posts/...`                       |
    | 2    | `feat: ...` | `src/components/...`, `src/hooks/...` |
    ```

## 4. 제안 시점 및 피드백 루프
- 사용자가 "커밋할 목록을 제시해줘" 또는 "커밋해줘"라고 요청할 때 이 스킬을 활성화합니다.
- 사용자가 특정 파일들을 제외하거나 그룹을 합치고 싶어 하면, 즉시 반영하여 다시 제안합니다.
